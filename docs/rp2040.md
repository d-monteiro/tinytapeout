# RP2040 Interface — NeuroPulse

The chip's FSM is fully deterministic. The RP2040 knows exactly which weight is needed at each chip clock cycle and pre-loads them into a PIO TX FIFO before each pass.

---

## Weight table

```c
// weights[post_neuron][pre_neuron]  —  4-bit values (0–15)
uint8_t weights[4][4] = {
    {0,  0,  0,  0},  // into neuron 0
    {10, 0,  0,  0},  // into neuron 1: w=10 from neuron 0
    {0,  0,  0,  0},  // into neuron 2
    {0,  0,  0,  0},  // into neuron 3
};
```

---

## FSM cycle map

One full pass = 28 chip clock cycles (7 states × 4 neurons):

```
Neuron 0: LATCH | TRACE | ACCUM_0 | ACCUM_1 | ACCUM_2 | ACCUM_3 | THRESH
Neuron 1: LATCH | TRACE | ACCUM_0 | ACCUM_1 | ACCUM_2 | ACCUM_3 | THRESH
Neuron 2: LATCH | TRACE | ACCUM_0 | ACCUM_1 | ACCUM_2 | ACCUM_3 | THRESH
Neuron 3: LATCH | TRACE | ACCUM_0 | ACCUM_1 | ACCUM_2 | ACCUM_3 | THRESH
```

During each `ACCUM_x` state, `uo_out[6]` (`in_accum`) is high and `uio_out[6:5]` shows `syn_sel`.
The RP2040 must have `weight[N][syn]` stable on `ui_in[3:0]` **before the rising edge** of that cycle.

---

## PIO weight-streaming

Pre-load the 16 weights for one pass (in FSM order) into the PIO TX FIFO:

```
Pass order: w[0][0], w[0][1], w[0][2], w[0][3],
            w[1][0], w[1][1], w[1][2], w[1][3],
            w[2][0], w[2][1], w[2][2], w[2][3],
            w[3][0], w[3][1], w[3][2], w[3][3]
```

The PIO program clocks each weight out in sync with `in_accum`:

```pio
; NeuroPulse weight streamer
; Side-set: ui_in[3:0] driven directly
; Waits for in_accum (uo_out[6]) then outputs next weight

.program weight_stream
    pull block          ; load next word (4 weights packed)
    set x, 3            ; 4 ACCUM states per neuron
loop:
    wait 1 gpio 22      ; wait for in_accum high (uo_out[6] → GPIO 22)
    out pins, 4         ; put 4-bit weight on ui_in[3:0]
    wait 0 gpio 22      ; wait for in_accum low
    jmp x-- loop
```

> Adjust GPIO numbers to match your wiring.

---

## LTP weight update

On each rising edge, sample `uo_out[7]` (`ltp_pulse`) and `uo_out[5:4]` (`neuron_sel`).

```c
void poll_ltp(void) {
    uint32_t out = gpio_get_all();
    bool ltp     = (out >> LTP_PULSE_GPIO) & 1;
    uint8_t post = (out >> NEURON_SEL_GPIO) & 0x3;
    uint8_t syn  = (out >> SYN_SEL_GPIO) & 0x3;   // uio_out[6:5]

    if (ltp) {
        if (weights[post][syn] < 15)
            weights[post][syn]++;
        // reload PIO FIFO with updated weights for next pass
        reload_pio_fifo();
    }
}
```

`ltp_pulse` is valid for exactly one chip clock cycle (during the THRESH state of the firing neuron).

## External spike injection

`ui_in[7:4]` is latched at the LATCH state of neuron 0 (once per pass).
To inject a spike into neuron `n`, set bit `n` of `ui_in[7:4]` for at least one full pass.

```c
// Hold spike on neuron 0 for one pass
gpio_put_masked(EXT_SPIKE_MASK, 0b0001 << EXT_SPIKE_BASE_GPIO);
sleep_us(1);   // >> 28 cycles at 50 MHz
gpio_put_masked(EXT_SPIKE_MASK, 0);
```

## Pin wiring summary

| RP2040 GPIO | Chip pin     | Direction | Description                     |
|-------------|--------------|-----------|----------------------------------|
| 0–3         | ui_in[3:0]   | RP→Chip   | Weight data (during ACCUM)       |
| 4–7         | ui_in[7:4]   | RP→Chip   | External spike injection         |
| 8           | uio_in[0]    | RP→Chip   | learn_ena                        |
| 16–19       | uo_out[3:0]  | Chip→RP   | Spike outputs                    |
| 20–21       | uo_out[5:4]  | Chip→RP   | neuron_sel                       |
| 22          | uo_out[6]    | Chip→RP   | in_accum (gate for weight write) |
| 23          | uo_out[7]    | Chip→RP   | ltp_pulse                        |
| 24–25       | uio_out[6:5] | Chip→RP   | syn_sel (during ACCUM)           |
| 26–29       | uio_out[3:0] | Chip→RP   | vmem[5:2] debug                  |
